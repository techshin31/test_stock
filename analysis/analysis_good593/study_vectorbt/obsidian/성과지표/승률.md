# 승률 · 거래 횟수 (Win Rate · Trade Count)

> 분류: 성과지표 | 관련: [[성과지표/샤프비율]] · [[성과지표/MDD]] · [[최적화/Grid_Search_최적화]]

---

## 개념

### 승률 (Win Rate)
전체 거래 중 **수익이 난 거래의 비율**이다.

```
10번 거래 중 7번 수익 → 승률 70%
```

### 거래 횟수 (Trade Count)
백테스트 기간 중 **진입·청산이 완료된 거래의 수**다.

---

## 계산 방법

```python
win_rate    = pf.trades.win_rate()    # 수익 거래 / 전체 거래
trade_count = pf.trades.count()       # 전체 완료 거래 수
```

---

## 승률만으로 판단하면 안 되는 이유

승률이 높아도 손실이 수익보다 크면 전체적으로 손해다.

```
승률 80%, 평균 수익 1%, 평균 손실 10%
→ 기댓값 = 0.8 × 1% + 0.2 × (-10%) = -1.2%  → 손실 전략
```

승률은 반드시 **손익비(평균 수익 / 평균 손실)**와 함께 봐야 한다.

---

## 거래 횟수의 중요성

거래 횟수가 너무 적으면 통계적 신뢰도가 낮다.

| 거래 횟수 | 신뢰도 |
|----------|--------|
| < 10회 | 매우 낮음 — 우연일 가능성 높음 |
| 10 ~ 30회 | 낮음 |
| 30 ~ 100회 | 보통 |
| > 100회 | 통계적으로 의미 있음 |

Grid Search에서 거래 횟수가 너무 적은 파라미터는 과최적화 가능성이 높다.

## 장점 / 단점

| | 내용 |
|---|---|
| **장점** | 직관적이고 이해하기 쉬움 |
| **장점** | 파라미터별 거래 활성도 비교에 유용 |
| **단점** | 승률 단독으로는 전략의 수익성을 알 수 없음 |
| **단점** | 거래 횟수가 적으면 승률 자체가 통계적으로 무의미 |

| 성향 | 역할 |
|------|------|
| 위험중립형 | 보조 참고용. 하락 방어 구조상 거래 자체가 적어 통계적 의미가 약함 |
| 적극투자형 | [[성과지표/Profit_Factor]]로 대체 권장. 인버스 ETF 신호 품질 평가에는 승률보다 손익비가 중요 |

**자동매매에서는 승률 단독 사용을 지양하고 반드시 Profit Factor와 함께 본다.**

## 이 시스템에서의 역할

`optimizer.py::grid_search()` 결과 테이블에 포함되어
파라미터별 거래 빈도와 승률을 함께 확인하는 데 사용된다.

거래 횟수가 극단적으로 적은 파라미터(< 5회)는
Sharpe가 높아도 신뢰하기 어려우므로 제외를 검토한다.
