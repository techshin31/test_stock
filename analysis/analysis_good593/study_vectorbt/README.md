### 06번 · 시장 국면 판별 전략

**파일**: `06_시장국면_판별전략.ipynb`

#### 선택 이유
모든 전략의 기반이 되는 **4국면 판별 로직**을 먼저 이해하기 위해 설계했다.
단순 이동평균 크로스 전략은 횡보장에서 손실이 크다는 문제를 해결하려면,
지금이 어떤 장인지를 먼저 알아야 한다.

#### 4국면 정의

| 국면 | 조건 | 포지션 |
|------|------|--------|
| UPTREND | MA20 > MA60 > MA120 AND ADX > 25 | 100% 매수 |
| DOWNTREND | MA20 < MA60 < MA120 AND ADX > 25 | 0% 전량 청산 |
| SIDEWAYS | ADX < 20 | 50% (볼린저 밴드) |
| TRANSITION | 위 3가지 외 | NaN 유지 (기존 포지션 그대로) |

#### 조심해야 할 것
- ADX는 **후행 지표**다. 추세가 시작된 후에야 ADX가 올라간다 → 진입이 늦어질 수 있음
- TRANSITION 국면이 길어지면 수익 기회를 놓칠 수 있음
- 판별 우선순위: SIDEWAYS → UPTREND → DOWNTREND → TRANSITION (순서가 결과에 영향)

### 07번 · MA정렬 + ADX 통합 전략

**파일**: `07_MA정렬_ADX_통합전략.ipynb`

#### 선택 이유
06번에서 정의한 4국면을 실제 매매에 적용한 첫 번째 전략이다.
국면별로 다른 전략을 자동으로 스위칭한다.

- UPTREND → 골든크로스 진입 (100%)
- SIDEWAYS → 볼린저 밴드 하단 반등 진입 (50%)
- DOWNTREND → 전량 청산
- TRANSITION → 신규 진입 차단, 기존 포지션 유지

#### 장점
- 하락장에서 현금을 보유하므로 MDD가 낮음
- 횡보장에서도 볼린저 밴드로 수익 기회를 만듦

#### 조심해야 할 것
- 청산은 DOWNTREND 진입 시에만 발생한다 (손절 없음)
- 강한 상승장에서 TRANSITION 구간이 길면 수익을 일부 반납함
- `from_orders(size_type="targetpercent")`에서 NaN = 포지션 유지, 0 = 전량 청산

### 08번 · Walk-Forward 최적화

**파일**: `08_워크포워드_최적화.ipynb`

#### 선택 이유
단순 백테스트는 전체 기간 데이터로 파라미터를 최적화하므로 **미래 데이터를 미리 본** 셈이 된다.
Walk-Forward는 이를 방지하기 위해 훈련/적용 구간을 시간 순서대로 분리한다.

#### 설계 원칙
```
훈련: 최근 12개월 → 최적 ADX threshold 탐색
적용: 다음  6개월 → 탐색된 파라미터 사용
       ↓ 6개월 후 재최적화 반복
```
시장 국면이 약 6개월 단위로 변한다는 가정 하에 설계됐다.

#### 장점
- 과최적화(Overfitting) 방지
- 시장 변화에 따라 파라미터가 자동으로 갱신됨
- 실전 운용 구조와 동일

#### 조심해야 할 것
- 훈련 구간이 짧으면 국면 다양성 부족 → 신뢰도 낮음
- 최적화 대상이 `adx_threshold` 하나뿐 → 다른 파라미터는 고정
- 테스트 구간 성과가 낮으면 전략 자체의 문제일 수 있음 (파라미터 탓이 아님)

### 09번 · 부분 매수/매도 전략

**파일**: `09_부분매수매도_전략.ipynb`

#### 선택 이유
07번은 한 번에 100% 매수/청산한다. 실전에서는 한 번에 전액 투자하기 어렵고,
분할 매수로 평균 단가를 낮추고 분할 매도로 수익을 보호하는 것이 더 현실적이다.

#### 3단계 분할 매수

| 단계 | 조건 | 목표 비중 |
|------|------|----------|
| 1차 | UPTREND + 골든크로스 | 40% |
| 2차 | UPTREND + MA20 지지 확인 (60일 이내) | 70% |
| 횡보 | SIDEWAYS + BB 하단 터치 | 30% |

#### 분할 매도

| 단계 | 조건 | 유지 비중 |
|------|------|----------|
| 1차 익절 | UPTREND → TRANSITION 전환 첫날 | 40% |
| 2차 청산 | 데드크로스 발생 | 10% |
| 전량 청산 | DOWNTREND 진입 | 0% |
| 횡보 청산 | SIDEWAYS + BB 상단 터치 | 0% |

#### 장점
- 진입/청산을 단계적으로 나눠 평균 단가 개선
- 추세가 꺾이는 초기 신호(TRANSITION)에서 일부 먼저 익절
- MACD, OBV 같은 복잡한 지표 없이 GC / MA20 / BB만 사용 → 신호 명확

#### 조심해야 할 것
- 단계별 청산마다 수수료 발생 (3번 청산 = 3번 수수료)
- 10% 잔량은 금액이 작아 실전에서 의미 없을 수 있음
- DOWNTREND 판별이 늦으면 고점 대비 -10~15% 손실 후 청산될 수 있음

### 10번 · 멀티종목 포트폴리오 백테스팅

**파일**: `10_포트폴리오_백테스팅.ipynb`

#### 선택 이유
09번 전략을 삼성전자 1종목에서 **5개 섹터 대표주**로 확장한다.
특정 종목이 하락장에 빠져도 다른 종목이 수익을 낼 수 있어 분산 효과가 생긴다.

#### 포트폴리오 구성

| 종목 | WICS 섹터 |
|------|----------|
| 삼성전자 | IT (반도체) |
| SK하이닉스 | IT (메모리) |
| NAVER | 경기관련소비재 (인터넷) |
| 현대차 | 경기관련소비재 (자동차) |
| POSCO홀딩스 | 소재 (철강) |

#### 국면별 모멘텀 비례 가중치

단순 1/N 균등 배분 대신, **모멘텀 강도에 비례해** 비중을 배분한다.

```
최종 비중 = 신호 크기 × (종목 모멘텀 / 유효 종목 모멘텀 합)
```

모멘텀 윈도우는 시장 국면에 따라 자동 선택된다:

| 국면 | 모멘텀 윈도우 | 이유 |
|------|------------|------|
| UPTREND | 126일 (6개월) | 추세가 길게 이어지므로 장기 모멘텀 신뢰도 높음 |
| TRANSITION | 63일 (3개월) | 방향 불확실, 중기 모멘텀으로 중간값 사용 |
| SIDEWAYS | 21일 (1개월) | 단기 등락 반복, 빠른 반응 필요 |
| DOWNTREND | 진입 안 함 | 전량 청산 국면 |

#### 가변 투자 비중
```
신호 많고 모멘텀 강한 날 → 투자 비중 증가 (현금 비중 감소)
신호 없거나 모멘텀 약한 날 → 현금 대기
음수 모멘텀 종목 → 신호 발생해도 진입 제외
```

#### 장점
- 종목 분산으로 MDD 감소
- 모멘텀 기반 비중 배분으로 수익 가능성 높은 종목에 자본 집중
- 현금 보유 비중이 시장 상황에 따라 자연스럽게 조절됨

#### 조심해야 할 것
- `cash_sharing=True`: 5종목이 현금 풀을 공유 → 동시에 여러 종목에 진입 시 현금 부족 가능
- 모멘텀이 모두 양수인 강세장에서는 진입 종목이 늘어 개별 비중이 줄어듦
- 단기 모멘텀(21일)은 노이즈에 민감 → SIDEWAYS 국면에서 잦은 신호 발생 주의
