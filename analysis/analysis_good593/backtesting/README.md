# 백테스팅 시스템 요구사항 — 탑다운(Top-Down) 전략

## 1. 전략 개요

**글로벌 매크로 → WICS 섹터 → 개별 종목** 순서로 투자 대상을 좁혀가는 3단계 탑다운 백테스팅.

```
[1단계] 매크로 국면 판단           [2단계] 섹터 선택                [3단계] 종목 선택
─────────────────────           ─────────────────────           ─────────────────────
data/asset/ CSV                 data/wics/ ZIP/CSV              data/재무제표/ CSV
                                                                + data/wics/ (주가 역산)

구리·BDI  → 경기 국면           Risk-On  → IT·산업재 비중↑      FA 7지표 점수 상위 N개
CPI·TNX   → 금리 환경           Risk-Off → 필수소비재·헬스케어↑   OPM·성장·부채·현금흐름
금·DXY    → 리스크 온/오프      고금리   → 에너지·소재 비중↑     PER·배당·생존성
```

**리밸런싱 주기**: 분기 1회 (DART 공시 주기에 맞춤)

## 2. 데이터 소스

### 2-1. 매크로 데이터 (`data/asset/`)

| 자산 | 티커 | 역할 | 투자 가능 여부 |
|------|------|------|--------------|
| 구리 (Copper) | `copper` | 경기 선행지표 | 신호 + 직접 투자 가능 |
| 금 (Gold) | `gold` | 리스크 온/오프 | 신호 + 직접 투자 가능 |
| 달러 인덱스 | `dxy` | 달러 강/약세 | 신호 전용 |
| 반도체 지수 (SOX) | `sox` | IT 경기 선행 | 신호 + ETF 추종 가능 |
| WTI 원유 | `wti` | 에너지 국면 | 신호 + 직접 투자 가능 |
| 해운 운임 (BDI) | `bdry` | 글로벌 교역량 | **신호 전용 (직접 투자 불가)** |
| 미국 CPI | `cpi` | 인플레이션 | **신호 전용** |
| 미국 10년 국채 | `tnx` | 금리 환경 | 신호 전용 |

- 컬럼: `Date, Open, High, Low, Close, Volume` (일봉, CPI는 `Date, Close, CPI_YoY`)
- 연도별 파일 분리 → 다년도 시 병합 필요

### 2-2. 섹터 및 종목 데이터 (`data/wics/`)

| 파일 | 내용 |
|------|------|
| `wics_company_YYYY.zip` | 2021~2025년 일별 WICS 분류 + 시가총액 |
| `wics_company_2026.csv` | 2026년 최신 데이터 |

**주요 컬럼**

| 컬럼 | 설명 |
|------|------|
| `CMP_CD` | 종목코드 |
| `CMP_KOR` | 종목명 |
| `IDX_CD` | WICS 세부 지수 코드 |
| `SEC_CD` | WICS 섹터 코드 (G10~G55) |
| `SEC_NM_KOR` | 섹터명 |
| `MKT_VAL` | 시가총액 (백만원) |
| `APT_SHR_CNT` | 적용 주식수 |
| `INFO_TRD_AMT` | 거래대금 |
| `DATE` | 날짜 (YYYYMMDD) |

**종가 역산 공식**
```
종가(원) = MKT_VAL(백만원) × 1,000,000 / APT_SHR_CNT
```

- 거래일 기준 일봉, 연간 약 244거래일
- 전체 종목 수: 약 2,600개 (KOSPI+KOSDAQ)
- 주의: Close만 역산 가능 (Open/High/Low 없음) → 종가 기준 거래만 시뮬레이션

### 2-3. 재무제표 데이터 (`data/재무제표/`)

| 파일 | 내용 |
|------|------|
| `income_statement_YYYY.csv` | 손익계산서 (매출, 영업이익, 순이익) |
| `balance_sheet_YYYY.csv` | 대차대조표 (자산, 부채, 자본) |
| `cash_flow_YYYY.csv` | 현금흐름표 (영업·투자·재무 CF) |
| `dart_company_2026.csv` | 종목코드(CMP_CD) ↔ DART코드(DART_CD) 매핑 |

**주요 컬럼**: `corp_code, bsns_year, reprt_code, account_id, account_nm, thstrm_amount`

- `reprt_code`: 11011=연간, 11012=Q1, 11013=반기, 11014=Q3
- 공시 지연: 결산 후 **45~60일 뒤** 실제 사용 가능 → 룩어헤드 바이어스 주의

## 4. 탑다운 3단계 전략 로직

### 1단계: 매크로 국면 판단 (매월 말 실행)

아래 신호를 조합해 **4가지 국면** 중 하나로 분류한다.

```
국면 A — Risk-On  + 저금리:  구리↑ & BDI↑ & TNX<4% & CPI<3%
국면 B — Risk-On  + 고금리:  구리↑ & BDI↑ & (TNX≥4% or CPI≥4%)
국면 C — Risk-Off + 저금리:  구리↓ or BDI↓ & TNX<4%
국면 D — Risk-Off + 고금리:  구리↓ or BDI↓ & (TNX≥4% or CPI≥4%)
```

**신호 계산 방식**
- 구리·BDI: 3개월 이동평균 기울기 (양수 = 상승 추세)
- TNX: 직전 월말 종가
- CPI: CPI_YoY 직전 발표값 (룩어헤드 바이어스 방지 → **익월 적용**)

### 2단계: 섹터 비중 결정 (국면별 포트폴리오)

| 국면 | 비중 확대 섹터 | 비중 축소 섹터 |
|------|--------------|--------------|
| A (Risk-On + 저금리) | IT(G45), 경기소비재(G25), 산업재(G20) | 유틸리티(G55), 필수소비재(G30) |
| B (Risk-On + 고금리) | 에너지(G10), 소재(G15), 금융(G40) | IT(G45), 경기소비재(G25) |
| C (Risk-Off + 저금리) | 헬스케어(G35), 유틸리티(G55), 필수소비재(G30) | 에너지(G10), 산업재(G20) |
| D (Risk-Off + 고금리) | 필수소비재(G30), 헬스케어(G35), 금(gold) 신호 참고 | IT(G45), 경기소비재(G25), 소재(G15) |

### 3단계: 종목 선택 — FA 7지표 점수 (선택된 섹터 내)

FA 7지표 각각을 점수화(0~2점)하여 합산, 상위 N개 종목 균등 편입.

| # | 지표 | 계산 방법 | Green(2점) | Yellow(1점) | Red(0점) |
|---|------|----------|-----------|------------|---------|
| 1 | 수익성 (OPM) | 영업이익 / 매출액 | ≥10% | 5~10% | <5% |
| 2 | 성장성 | YoY 매출 성장률 | ≥10% | 0~10% | <0% |
| 3 | 재무안정성 | 부채 / 자본 | <100% | 100~200% | >200% |
| 4 | 현금흐름 | 영업CF / 순이익 | >1.0 | 0~1.0 | <0 |
| 5 | 밸류에이션 | PER (섹터 중앙값 대비) | <중앙값×0.8 | 중앙값±20% | >중앙값×1.5 |
| 6 | 주주환원 | 배당수익률 | ≥3% | 1~3% | <1% |
| 7 | 재무생존성 | 순적자 기업 현금잔고/월평균지출 | >24개월 | 12~24개월 | <12개월 |

- **PER 계산**: 종가(WICS 역산) / EPS(순이익 / 주식수)
- **최소 종목 수**: 섹터당 3개 이상 편입 (분산 목적)
- **최대 종목 수**: 섹터당 상위 10개 이내

## 5. 백테스팅 실행 파라미터

### 5-1. 기간 필터링

연도 / 분기 / 월 단위로 기간 지정 가능.

```python
# 연도 전체
run_backtest(year=2023)

# 특정 분기
run_backtest(year=2023, quarter=3)   # 7월~9월

# 특정 월
run_backtest(year=2023, month=11)

# 다년도
run_backtest(year=[2022, 2023, 2024])
```

| 분기 | 기간 |
|------|------|
| Q1 | 01-01 ~ 03-31 |
| Q2 | 04-01 ~ 06-30 |
| Q3 | 07-01 ~ 09-30 |
| Q4 | 10-01 ~ 12-31 |

### 5-2. 기본 설정값

| 파라미터 | 기본값 | 설명 |
|---------|--------|------|
| `initial_cash` | 100,000,000원 | 초기 자본금 |
| `commission` | 0.015% | 매매 수수료 (키움 기준) |
| `slippage` | 0.05% | 슬리피지 |
| `rebalance_freq` | `quarter` | 리밸런싱 주기 |
| `top_n_stocks` | 5 | 섹터당 편입 종목 수 |
| `sector_weights` | 국면별 동적 | 섹터 배분 비율 |

## 6. 성과 지표

| 지표 | 설명 |
|------|------|
| **CAGR** | 연환산 수익률 |
| **MDD** | 최대 낙폭 |
| **Sharpe Ratio** | 위험 대비 초과 수익 (무위험이자율: TNX 사용) |
| **Sortino Ratio** | 하방 리스크 기준 수익 |
| **Calmar Ratio** | CAGR / MDD |
| **Hit Rate** | 수익 종목 비율 |
| **월별 수익률 히트맵** | Jupyter 시각화용 |

**벤치마크**: KOSPI 전체 시가총액 가중 포트폴리오 (WICS 전 종목 MKT_VAL 비중)

## 8. 구현 순서

1. `loader_wics.py` — 종가 역산 및 날짜 필터 검증
2. `loader_dart.py` — 공시 지연(+60일) 적용한 FA 지표 추출
3. `loader_asset.py` — 매크로 신호 계산 검증
4. `macro_signal.py` — 국면 분류 백테스트 사후 검증 (2021~2025)
5. `sector_selector.py` — 국면별 섹터 수익률 비교
6. `stock_scorer.py` — FA 점수 기반 종목 선택 검증
7. `runner.py` — 3단계 통합 E2E 실행
8. `metrics.py` + `report.py` — KOSPI 대비 성과 비교

## 9. 주의사항

| 항목 | 내용 |
|------|------|
| **룩어헤드 바이어스** | DART 재무제표는 결산 후 60일 뒤 적용 (예: 2023 Q3 → 2023-11-15 이후 사용) |
| **CPI 지연** | 발표월 CPI는 익월부터 신호로 사용 |
| **BDI 직접 투자 불가** | 신호 생성 전용, 포트폴리오 편입 불가 |
| **종가 한계** | WICS 역산 주가는 Close만 가능 → Open/High/Low 없음, 장 마감 체결 가정 |
| **서바이버십 바이어스** | 상장폐지 종목이 과거 WICS 데이터에서 제거되어 있을 수 있음 |
| **월 단위 신뢰도** | 거래일 약 20일, 통계적 신뢰도 낮아 참고용으로만 활용 |
