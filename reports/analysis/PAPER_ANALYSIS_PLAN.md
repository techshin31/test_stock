# PAPER 전체 분석 계획

- 작성 시각: 2026-07-22 10:05 KST
- 대상: `PAPER / aggressive / ***9904-01`
- 최초 원금 가정: 500,000,000원, 입출금 없음
- 목적: 원금 손실 원인을 복원하고, 실제 PAPER에서 비용 차감 후 수익률을 개선할 변경 후보를 결정한다.
- 비목표: 이 단계에서는 전략 파라미터, 주문 설정, 스케줄러, 실행 모드를 변경하지 않는다. REAL은 분석·실험 대상에서 제외한다.

## 1. 현재 판단

현재 상태는 전략을 바로 튜닝할 단계가 아니다. 아래 세 문제를 순서대로 해결해야 한다.

1. 최초 원금 5억과 공식 기준선 463,025,590원이 달라 전체 손실이 공식 수익률에서 누락된다.
2. 2026-07-09의 체결금액은 1,953,105,060원으로 원금의 3.91배였고, 240개 주문 중 192개가 거절됐다.
3. 연구 리플레이와 실제 PAPER 실행 경로가 다르다. 연구는 일봉 종가·비례 체결을 사용하지만 PAPER에는 분할체결, 주문 불명 결과, 위험청산 실패가 존재한다.

따라서 분석의 핵심 질문은 “어떤 파라미터가 수익을 높이는가”가 아니라 다음과 같다.

- 원금 대비 약 3천만 원 손실은 언제, 어떤 종목과 주문에서 발생했는가?
- 손실은 종목 선택, 포지션 크기, 잦은 리밸런싱, 주문 실패 중 어디에 가장 크게 귀속되는가?
- 연구 백테스트의 알파가 실제 PAPER 주문 조건과 비용을 적용해도 남는가?
- 같은 신호에서 위험을 줄여도 순수익과 벤치마크 초과수익을 유지할 수 있는가?

## 4. 기존 연구 결과의 해석

### 4.1 서로 다른 연구 결과

- 비중 배분만 분리한 38개월 연구에서는 동일 비중이 Sharpe 1.371로 가장 높았다.
- 통합 리플레이 전체 757거래일에서는 FA 초과점수 비례가 Sharpe 0.793으로 가장 높았지만, 39개 월별 실행 중 32개가 WARNING이었고 MDD는 -23.27%, 누적 비용률은 11.45%였다.
- PASS/PUBLISHED만 사용한 107거래일에서는 FA 초과점수 비례가 Sharpe 1.216으로 가장 높았지만, 연 변동성 45.76%, MDD -16.99%였다.

### 4.2 현재 가설

현재 PAPER는 짧고 변동성이 큰 PASS 구간의 승자를 선택한 상태다. 이 결과는 수익 가능성을 보여주지만 원금 방어 전략이라는 증거는 아니다. 실제 PAPER의 -5.95%는 연구의 -16.99% MDD 범위 안에 있으므로 코드 오작동만으로 설명할 수 없고, 전략 자체의 위험 수준도 원인 후보로 유지해야 한다.

## 5. 분석 단계

### 단계 A — 원금 손익 원장 복원

작업:

1. 5억 원 시작 시각과 입출금·수동 주문 여부를 확정한다.
2. KIS, DB, 파일 스냅샷을 분 단위로 정렬한다.
3. 일별 `현금 + 보유평가액 + 미수/정산항목 = 총자산`을 대사한다.
4. 평균단가 방식으로 종목별 실현·미실현 손익을 재구성한다.
5. 미확정 차이를 `누락 체결`, `입출금`, `비용`, `가격 차이`, `설명 불가`로 분리한다.

산출물:

- `inception_nav_daily.csv`
- `trade_ledger_reconciled.csv`
- `pnl_attribution_daily.csv`
- `reconciliation_gaps.csv`

완료 기준:

- 모든 거래일의 총자산 대사 결과가 존재한다.
- 설명 불가 잔액은 5억 원의 0.1% 이하이거나 원천 누락 사유가 명시된다.
- 모든 FILLED 주문은 체결 수량과 연결되고, UNKNOWN/REJECTED 전환 이력이 보존된다.

### 단계 B — 손실 기여도 분해

분해 축:

- 종목별: 실현손익, 미실현손익, 비용, 보유기간, 최대 역행폭
- 주문 사유별: 신규 진입, 비중 확대, 비중 축소, TA 청산, hard stop, trailing stop
- 시점별: 일자, 장중 구간, 시장 국면
- 실행별: 정상 체결, 분할체결, 재주문, 불명 결과, 거절, 가격가드

핵심 답변:

- 총손실 중 상위 5개 종목·주문이 차지하는 비중
- 7월 9일 주문 폭증이 손익과 비용에 미친 영향
- 손절·트레일링 규칙이 손실을 줄였는지, 반등 전에 손실을 확정했는지
- 현재 수익 종목이 과거 손실을 얼마나 상쇄했는지

### 단계 C — 실행 품질 분석

측정 항목:

- 주문 의도 → 제출 → 접수 → 체결 → 계좌 반영 전환율
- 주문 불명률과 최종 확정 시간
- 위험청산 성공률과 완료 시간
- 동일 종목·동일 방향 중복 주문률
- 일일 체결금액/기초자산, 종목별 재주문 횟수
- 주문 예상가 대비 체결가 차이와 비용 드래그

운영 게이트:

- 위험청산 성공률 100%
- 평가기간의 미확정 주문 0건
- 동일 의사결정에 대한 중복 체결 0건
- 일일 회전율 상한은 검증된 리플레이 분포의 상위 분위수로 결정하며, 7월 9일 391%는 즉시 경보 구간으로 분류한다.

### 단계 D — 전략·실행 동일성 리플레이

현재 PAPER의 실제 경로인 `FaTaMomentumStrategy`와 `LiveTrader`를 기준으로 재현한다.

반드시 포함할 조건:

- 당시 시점에 발행된 FA 데이터만 사용
- 당시 완결 일봉과 시장 국면만 사용
- 정수 주식 수, 현금 버퍼, 15% 상한, 리밸런싱 괴리 적용
- 수수료·세금·종목 규모별 슬리피지 반영
- 주문 거절·불명 결과·분할체결을 별도 실행 시나리오로 반영
- 동일 신호가 PAPER와 리플레이에서 같은 목표 비중과 주문 사유를 내는지 비교

비교 후보는 과최적화를 막기 위해 다섯 개로 제한한다.

| 후보 | 신호 | 비중 | 실행 변경 | 목적 |
|---|---|---|---|---|
| A | 현재 FA+TA | FA 초과점수, 종목당 15% | 현재 방식 | 기준 전략 |
| B | 현재 FA+TA | 동일 비중 | 현재 방식 | FA 비중 효과 분리 |
| C | 현재 FA+TA | FA 초과점수, 종목당 8~10% | 현재 방식 | 집중도 축소 효과 |
| D | 현재 FA+TA | 현재 비중 | 넓은 리밸런싱 밴드·일 1회 | 회전율 효과 |
| E | 현재 FA+TA | 변동성 조정 비중 | 강화된 포트폴리오 손실 게이트 | 위험조정 효과 |

### 단계 E — 워크포워드와 최종 의사결정

검증 구조:

1. 학습·튜닝 구간과 최종 홀드아웃을 분리한다.
2. 시장 상승·하락·횡보 구간을 모두 포함한다.
3. 모든 후보에 같은 데이터, 비용, 체결 규칙을 사용한다.
4. 전체 결과뿐 아니라 각 OOS 윈도우의 분포를 비교한다.
5. 시험한 후보 수를 기록하고 PBO·Deflated Sharpe로 선택편향을 확인한다.

후보 승인 조건:

- 비용 차감 OOS 성과가 현재 전략보다 나쁘지 않다.
- MDD와 하방 변동성이 의미 있게 감소한다.
- 초과수익이 한 종목이나 한 기간에 과도하게 집중되지 않는다.
- 회전율·비용·주문 수가 실제 PAPER가 감당할 수 있는 수준이다.
- 전략 신호와 PAPER 주문 결과가 동일성 검사를 통과한다.

승인 후에도 먼저 shadow replay와 PAPER 관측을 진행하며 REAL은 기존 60거래일·사람 승인 게이트를 그대로 유지한다.

## 6. KPI 체계

### Primary outcomes

| KPI | 정의 | 판단 용도 |
|---|---|---|
| 최초 원금 기준 순수익률 | `(기말자산 - 순입출금) / 5억 - 1` | 실제 자본 회복 여부 |
| KOSPI 대비 비용 차감 초과수익 | 전략 순수익률 - 동일 기간 KOSPI 수익률 | 전략 선택의 가치 |
| 최대 낙폭 | 고점 대비 최저 NAV 하락률 | 원금 훼손 수준 |

### Diagnostic drivers

| KPI | 정의 |
|---|---|
| 종목·주문 사유별 손익 기여 | 실현 + 미실현 - 비용을 상호배타적으로 분해 |
| 회전율과 비용 드래그 | 체결금액/기초자산, 총비용/기초자산 |
| 승률·손익비·profit factor | 거래 품질과 꼬리손실 확인 |
| 신호→체결 괴리 | 목표 비중과 실제 체결 후 비중 차이 |

### Guardrails

| KPI | 목표 방향 |
|---|---|
| 잔고 대사 오차 | 0에 수렴, 설명 불가 0.1% 이하 |
| 위험청산 성공률 | 100% |
| 미확정 주문률 | 0% |
| 종목 집중도 | 후보별 상한 비교 후 결정 |
| 일일 손실·포트폴리오 MDD | 사용자가 허용할 위험 예산 안에서 제한 |

## 7. 수정 백로그의 예상 위치

분석 결과가 나온 뒤 다음 순서로 수정한다.

1. `core/analytics/trading_performance.py`: 최초 원금 기준 수익률·MDD·손익 대사 추가
2. `core/execution/trader.py`: 일일 주문 예산, 중복 리밸런싱 제한, 포트폴리오 고점 손실 게이트
3. `core/broker/kis_api.py` 및 주문 정산 경로: 위험청산 불명 결과 우선 조회·확정
4. `core/strategy/fa_ta_momentum.py`: 검증을 통과한 비중·손절 후보만 반영
5. `apps/backtester/fa_weighting_replay.py`, `core/backtest/engine.py`: 운영 동일성 리플레이와 홀드아웃 평가

`core/strategy/aggressive.py`는 현재 PAPER 실행 클래스가 아니므로 우선 수정 대상이 아니다.

## 8. 즉시 시작 순서

1. 단계 A의 원장 스키마와 대사 규칙을 확정한다.
2. 확보된 7월 9일 이후 데이터로 우선 복원하고, 원천이 없는 구간은 명시적인 gap으로 남긴다.
3. 단계 B·C를 같은 원장에서 계산해 손실 원인과 실행 결함을 분리한다.
4. 그 결과가 나온 뒤에만 단계 D의 다섯 후보를 실행한다.
5. 최종적으로 “운영 결함 수정”과 “전략 변경”을 별도 변경 세트로 나눈다.

현재 첫 작업 단위는 **원금 손익 원장 복원**이다. 이 원장이 완료되기 전에는 수익률 개선 파라미터를 선택하지 않는다.
