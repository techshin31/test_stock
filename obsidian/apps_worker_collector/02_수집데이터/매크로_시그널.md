# 매크로 시그널

관련 실행: [[../01_실행가이드/target_macro|target macro]]

## 한 줄 정의

매크로 시그널은 국내 종목을 고르기 전에 글로벌 경기, 금리, 원자재, 위험 선호, 환율, 제조업, 한류 수요를 읽기 위한 외부 시계열이다. Collector는 이 값을 `macro_signals`에 저장하고, Analyzer는 `available_date` 기준으로 과거 시점에 실제 사용 가능했던 값만 읽는다.

## 실제로 수집하는 데이터

수집 대상은 `data/preprocess/macro_signals.py`의 `_SIGNAL_META`와 `apps/worker/fa_contract.py`의 `MACRO_SIGNALS`에 정의된 17개 신호다.

| 코드 | 원천/티커 | 주기 | 트레이딩에서 보는 의미 |
|---|---|---|---|
| `COPPER` | Yahoo Finance `HG=F` | 일간 | 경기민감 원자재. 산업재, 소재, 경기순환 업종의 수요 기대를 보는 데 사용한다. |
| `GOLD` | Yahoo Finance `GC=F` | 일간 | 안전자산 선호와 실질금리 부담의 proxy다. 위험 회피 국면 판단에 보조적으로 쓴다. |
| `WTI` | Yahoo Finance `CL=F` | 일간 | 에너지 가격과 인플레이션 압력의 proxy다. 정유, 화학, 운송, 소비 업종 해석에 연결된다. |
| `TNX` | Yahoo Finance `^TNX` | 일간 | 미국 장기금리 방향. 성장주 할인율, 금융주, 달러 강세 환경 해석에 필요하다. |
| `US2Y` | Yahoo Finance `^IRX` | 일간 | 코드명은 2년물 금리지만 실제 구현 원천은 `^IRX`다. 단기금리 proxy로 쓰기 전 명칭과 원천 검증이 필요하다. |
| `CPI` | FRED `CPIAUCSL` | 월간 | 물가 압력. CPI는 FRED vintage를 수집해 최초 발표/수정 이력을 보존한다. |
| `SOX` | Yahoo Finance `^SOX` | 일간 | 반도체 사이클과 글로벌 기술주 위험 선호를 보는 핵심 proxy다. |
| `BDRY` | Yahoo Finance `BDRY` | 일간 | 건화물 운임 ETF다. BDI 대체 지표로 해운, 소재, 글로벌 물동량 proxy에 가깝다. |
| `VIX` | Yahoo Finance `^VIX` | 일간 | 시장 변동성/공포 지표. 위험 축소 국면을 감지하는 데 쓴다. |
| `DXY` | Yahoo Finance loader는 `DX-Y.NYB`, contract는 `DX=F` | 일간 | 달러 강세/약세. 원화, 외국인 수급, 수출주 환경 해석에 필요하다. 원천 키 불일치 검증이 필요하다. |
| `USDKRW` | Yahoo Finance `KRW=X` | 일간 | 원/달러 환율. 수출주, 외국인 수급, 환손익 민감도 판단에 중요하다. |
| `GPR` | Caldara & Iacoviello GPR Excel | 월간 | 지정학적 리스크. 방산, 에너지, 위험자산 회피 국면 해석에 사용한다. |
| `ISM_PMI` | FRED `NAPMPMI` | 월간 | 미국 제조업 경기 레벨. 글로벌 경기민감 업종의 선행 proxy로 쓴다. |
| `SEMIPROD` | FRED `IPG3344S` | 월간 | 반도체 및 전자부품 생산 지표. 반도체 업종의 실물 사이클 proxy다. |
| `GTREND_KPOP` | pytrends keyword `K-pop` | 월간 | 엔터, 미디어, 소비재 등 한류 수요 proxy다. |
| `GTREND_KDRAMA` | pytrends keyword `Korean drama` | 월간 | 콘텐츠, 미디어, 플랫폼, 소비재 수요 proxy다. |
| `KR_TOURIST` | KTO 관광 API | 월간 | 외국인 입국자 수. 면세, 화장품, 유통, 여행/레저 업종 수요 proxy다. |

## 트레이딩 입장에서 왜 필요한가

이 데이터는 개별 종목의 매수 신호를 직접 만들기보다, 어떤 업종을 먼저 볼지 결정하는 top-down 입력이다.

- `macro_signals`는 Analyzer의 매크로 분석에서 추세와 방향성을 계산하는 원천이다.
- 매크로 신호와 WICS 업종 수익률의 관계를 통해 유리한 업종 후보를 만든다.
- `available_date`를 저장하므로, 과거 분석 시점에 아직 발표되지 않은 데이터를 쓰는 look-ahead bias를 줄일 수 있다.
- CPI처럼 수정 발표가 있는 데이터는 `revision_no`를 보존해 point-in-time 분석이 가능하다.

## 수집 방식과 라이브러리 평가

| 영역 | 현재 방식 | 판단 |
|---|---|---|
| 시장 가격/지수 | `yfinance`로 Yahoo Finance 종가 수집 | 연구/프로토타입에는 적합하다. 운영 매매 시스템에서는 공급 안정성, 결측, 티커 변경, 조정가격 검증이 필요하다. |
| FRED 지표 | `requests`로 FRED API 또는 공개 CSV 조회 | 공식 원천이라 적절하다. CPI vintage 수집은 point-in-time 관점에서 특히 좋다. |
| Google Trends | `pytrends` 사용, 5년 단위 chunk 후 월간 평균 | 수요 proxy로는 유용하지만 비공식 라이브러리와 0-100 정규화 특성 때문에 장기 비교 안정성 검증이 필요하다. |
| KTO 관광 | `requests`로 공공데이터 API 호출 | 관광 수요 지표로 적절하다. API 응답 필드 변형에 대비한 파싱은 되어 있다. |
| GPR | 연구자 제공 Excel 다운로드 | 지표 성격에는 맞다. 파일 구조 변경과 배포 지연을 모니터링해야 한다. |

## 현재 구현에서 주의할 점

- `US2Y`라는 코드명은 실제 원천 `^IRX`와 의미가 다를 수 있다. 금리 스프레드나 2년물 민감도를 해석하기 전 원천 교정이 필요하다.
- `DXY`는 contract의 `source_value_key`와 실제 loader 티커가 다르다. 저장된 `source_value_key`만 보고 원천을 추적하면 혼동될 수 있다.
- CPI 외의 `SOURCE_RELEASE_DATE` 계열은 실제 과거 발표일을 원천에서 모두 복원하지 않고, 일부는 수집일을 사용한다. 역사적 백테스트에는 보수적이지만 실제 당시 사용 가능 시점과 다를 수 있다.
- `yfinance` 기반 일간 데이터는 공식 거래소 피드가 아니다. 실거래 의사결정 전에 결측률, 수정주가, 상장폐지/티커 변경 처리를 점검해야 한다.
- Google Trends는 장기 구간을 나눠 받아 붙이므로 chunk 간 스케일 비교가 왜곡될 수 있다.

## 생성 주기와 저장

| 항목 | 생성 주기 | Collector 저장 방식 |
|---|---|---|
| Yahoo 기반 가격/지수 | 거래일 단위 | 결측 제거 후 최대 5영업일 forward fill |
| FRED/GPR/KTO | 월간 | 월초 기준 `observation_date`로 정규화 |
| Google Trends | 일간 관심도 원천을 월간 평균 | 월초 기준 월간 시계열로 저장 |
| CPI | 월간, revision 존재 | FRED vintage별 `revision_no` 보존 |

저장 위치는 `macro_signals`다. 자세한 전처리와 upsert 방식은 [[../03_전처리_저장/macro_signals_전처리_저장|macro_signals 전처리 저장]]을 참고한다.
