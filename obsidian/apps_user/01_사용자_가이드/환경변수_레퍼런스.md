# 환경변수 레퍼런스

근거 코드: `apps/user/config.py`, `apps/user/.env.sample`

`load_config()`는 기본적으로 `apps/user/.env`를 읽는다. `QUANTPILOT_ENV_FILE`이 있으면 그 경로를 대신 읽는다.

```text
Path(env_file or os.getenv("QUANTPILOT_ENV_FILE", "apps/user/.env"))
```

이미 프로세스 환경 변수에 값이 있으면 `.env`가 덮어쓰지 않는다. `load_dotenv(..., override=False)` 동작 때문이다.

## 사용자 정보

| 변수 | 필수 | 기본값 | 저장/사용 위치 |
|---|---:|---|---|
| `USER_EMAIL` | 필수 | 없음 | `users.email` 조회/생성 기준 |
| `USER_DISPLAY_NAME` | 선택 | `None` | 신규 사용자 생성 시 `users.display_name` |

`USER_EMAIL`이 이미 있으면 `register`는 기존 사용자를 재사용한다. `is_active=False` 사용자도 현재 코드는 별도 차단 없이 조회 결과를 사용한다.

## 증권사 공통

| 변수 | 필수 | 기본값 | 의미 |
|---|---:|---|---|
| `BROKER_CODE` | 선택 | `KIS` | `user_broker_credentials.broker_code` |
| `KIS_APP_KEY` | 필수 | 없음 | 암호화 후 `api_key` 저장 |
| `KIS_APP_SECRET` | 필수 | 없음 | 암호화 후 `api_secret` 저장 |
| `KIS_ENV` | 선택 | `paper` | `real`이면 `REAL`, 그 외는 `PAPER` |

`apps/user`는 `KIS_ENV`를 소문자로 바꾼 뒤 `real`인지 확인한다.

```text
KIS_ENV=real -> environment_code=REAL
그 외 값      -> environment_code=PAPER
```

## 주식 계좌

| 변수 | 필수 | 기본값 | 저장 위치 |
|---|---:|---|---|
| `KIS_DOMESTIC_STOCK_ACCOUNT_NO` | 필수 | 없음 | `account_number` |
| `KIS_DOMESTIC_STOCK_ACCOUNT_PRODUCT_CODE` | 필수 | 없음 | `extra.account_product_code` |

주식 계좌는 항상 저장을 시도한다.

저장되는 `extra` 예시:

```json
{
  "account_product_code": "01",
  "account_type": "STOCK"
}
```

## 선물 계좌

| 변수 | 필수 | 기본값 | 저장 위치 |
|---|---:|---|---|
| `KIS_DOMESTIC_FUTURES_ACCOUNT_NO` | 선택 | `None` | `account_number` |
| `KIS_DOMESTIC_FUTURES_ACCOUNT_PRODUCT_CODE` | 선택 | `None` | `extra.account_product_code` |

선물 계좌는 두 값이 모두 있을 때만 저장된다.

저장되는 `extra` 예시:

```json
{
  "account_product_code": "03",
  "account_type": "FUTURES"
}
```

현재 Trader 초기화 경로는 `account_type=STOCK`을 조회한다. 선물 계좌 저장은 미래 확장 또는 별도 사용 경로를 위한 데이터다.

## 암호화

| 변수 | 필수 | 기본값 | 의미 |
|---|---:|---|---|
| `CREDENTIAL_ENCRYPTION_KEY` | 필수 | 없음 | `api_key`, `api_secret` 암복호화 |

키 생성 명령:

```bash
python -c "from core.utils.crypto import generate_key; print(generate_key())"
```

같은 DB에 저장된 자격증명을 계속 읽으려면 같은 키를 계속 사용해야 한다.

## PostgreSQL

| 변수 | 필수 | 기본값 | 의미 |
|---|---:|---|---|
| `POSTGRES_HOST` | 필수 | `localhost` | DB 호스트 |
| `POSTGRES_PORT` | 선택 | `5432` | DB 포트 |
| `POSTGRES_USER` | 필수 | 없음 | DB 사용자 |
| `POSTGRES_PASSWORD` | 필수 | 없음 | DB 비밀번호 |
| `POSTGRES_DB` | 필수 | `quantpilot_db` | DB 이름 |

`load_config()`의 필수 검증에는 `POSTGRES_PORT`가 포함되지 않는다. `build_db_config()`에서 없으면 `5432`를 사용한다.

## Trader와 공유해야 하는 값

Trader가 등록된 자격증명을 찾으려면 다음 값이 user 등록 시점과 trader 실행 시점에 맞아야 한다.

| 값 | User 등록 | Trader 조회 |
|---|---|---|
| 사용자 | `USER_EMAIL` | `USER_EMAIL` |
| 증권사 | `BROKER_CODE` | `BROKER_CODE` |
| 거래 환경 | `KIS_ENV -> environment_code` | `TraderConfig.environment_code` |
| 암호화 키 | `CREDENTIAL_ENCRYPTION_KEY` | `CREDENTIAL_ENCRYPTION_KEY` |

`ALLOW_LIVE_ORDER`는 `apps/user` 등록 판단에는 쓰이지 않는다. 실주문 허용 여부는 Trader 설정과 gate에서 확인한다.
