# 계좌와 거래환경 해석

`apps/user`는 투자 판단을 하지 않는다. 투자자 관점에서 이 앱은 “어떤 계좌와 어떤 거래 환경으로 QuantPilot Trader가 움직일 수 있는가”를 정하는 준비 단계다.

## 이 앱이 결정하는 것

| 항목 | 투자자 의미 |
|---|---|
| `USER_EMAIL` | 어떤 투자자/운영자 명의의 자격증명인지 식별 |
| `BROKER_CODE` | 어떤 증권사 API를 사용할지 식별 |
| `KIS_ENV` | 모의투자 자격증명인지 실계좌 자격증명인지 구분 |
| 주식 계좌번호 | Trader가 주식 주문/잔고 조회에 사용할 계좌 |
| API 키/시크릿 | 증권사 API 접근 권한 |
| `CREDENTIAL_ENCRYPTION_KEY` | DB에 저장된 API 키/시크릿을 다시 읽기 위한 복호화 키 |

## PAPER와 REAL

`KIS_ENV=real`이면 DB에는 `environment_code=REAL`로 저장된다. 그 외 값은 `PAPER`로 저장된다.

| 환경 | 의미 | 투자자 관점 |
|---|---|---|
| `PAPER` | 모의투자 | 전략과 운영 절차를 검증하는 기본 환경 |
| `REAL` | 실계좌 | 실제 자금과 주문 가능성이 있는 운영 환경 |

자격증명을 `REAL`로 등록하는 것만으로 주문이 나가지는 않는다. 주문 실행은 `apps/trader`의 초기화, live order gate, `ALLOW_LIVE_ORDER=true` 확인을 별도로 통과해야 한다.

## 등록과 주문의 차이

```text
apps/user register
  -> 사용자/자격증명 저장
  -> 주문 없음

apps/trader executor
  -> 등록된 STOCK 자격증명 조회
  -> KIS broker 생성
  -> gate 통과 후 주문 루프 진입
```

투자자에게 중요한 점은 자격증명 저장이 “계좌 연결 준비”이지 “투자 실행 승인”이 아니라는 것이다.

## 계좌 유형

현재 등록 도구는 두 계좌 유형을 저장할 수 있다.

| account_type | 저장 조건 | 현재 운영 의미 |
|---|---|---|
| `STOCK` | 주식 계좌 env 필수 | Trader가 조회하는 주 계좌 |
| `FUTURES` | 선물 계좌 env 2개가 모두 있을 때 | 저장은 가능하지만 현재 Trader 초기화의 기본 조회 대상은 아님 |

Trader는 `fetch_credential_by_account_type(..., account_type="STOCK")`로 주식 계좌를 찾는다. 따라서 실운영 전에는 `list` 출력에서 `type=STOCK`, `env=PAPER/REAL`이 의도와 맞는지 먼저 확인한다.

## 보안과 운영 리스크

| 리스크 | 해석 |
|---|---|
| 암호화 키 분실 | 저장된 API 키/시크릿 복호화 불가. DB 행이 있어도 Trader가 사용할 수 없음 |
| 다른 키로 실행 | 복호화 실패 또는 잘못된 운영 환경 구성 |
| 계좌번호 평문 저장 | DB 접근 권한 관리가 중요함. 출력은 마스킹되지만 DB 컬럼은 평문 |
| `USER_EMAIL` 불일치 | Trader가 등록된 사용자를 찾지 못함 |
| `KIS_ENV` 불일치 | PAPER에 등록하고 REAL로 조회하거나 반대 상황이 발생 |

## 투자자 확인 체크리스트

1. `python -m apps.user list`에서 사용자 이메일이 맞는지 확인한다.
2. `type=STOCK` 자격증명이 있는지 확인한다.
3. `env=PAPER` 또는 `env=REAL`이 현재 운용 의도와 맞는지 확인한다.
4. 실계좌 운용 전에는 Trader 쪽 `ALLOW_LIVE_ORDER`와 gate 로그를 별도로 확인한다.
5. API 키 교체 후에는 같은 계좌 조합으로 `register`를 다시 실행해 갱신한다.

## Collector/Analyzer와의 관계

Collector와 Analyzer는 시장 데이터와 투자 후보를 만든다. `apps/user`는 그 투자 후보를 실제 계좌로 실행할 수 있게 만드는 인증 데이터 준비 단계다.

```text
Collector -> Analyzer -> universe
                         |
apps/user -> credentials |
                         v
                     Trader
```

투자 후보 품질과 계좌 연결 안전성은 서로 다른 문제다. Analyzer 결과가 좋아도 자격증명이나 거래 환경이 맞지 않으면 Trader는 시작하지 못한다.
