# 일일 운영 체크리스트

근거 코드: `apps/trader/__main__.py`, `apps/trader/planner.py`, `apps/trader/runner.py`, `apps/trader/monitor.py`

## 장전 이전

1. PostgreSQL이 실행 중인지 확인한다.
2. `apps/user register`로 같은 `USER_EMAIL`, `BROKER_CODE`, `KIS_ENV`의 STOCK 자격증명이 등록되어 있는지 확인한다.
3. Analyzer publish가 끝나 `universe`에 ACTIVE 종목이 있는지 확인한다.
4. `apps/trader/.env`의 `STRATEGY_NAME`이 활성 전략 이름과 맞는지 확인한다.
5. 실계좌라면 `KIS_ENV=real`, `ALLOW_LIVE_ORDER=true`를 의도적으로 설정했는지 확인한다.

## 08:30 planner

```bash
python -m apps.trader planner
```

확인할 출력:

| 출력 | 확인 의미 |
|---|---|
| `[TRADER] KIS_ENV=...` | 의도한 환경과 계좌인지 확인 |
| `[PLANNER] 포지션 동기화` | KIS 잔고를 `positions`에 반영 |
| `[PLANNER] SELL_ONLY 등록` | 보유 중이지만 오늘 universe 밖인 종목을 청산 대상으로 추가 |
| `[PLANNER] 기업 위험 매수차단` | `company_risk_states`에 의해 신규 매수가 막힌 종목 |
| `[PLANNER] trade_plans 생성 완료` | 오늘의 실행/스킵 계획 저장 완료 |

`universe`가 비어 있으면 운영 모드에서는 계획을 만들지 않는다. 테스트가 목적이면 `planner --test`를 쓴다.

## 09:00-15:20 executor

```bash
python -m apps.trader executor
```

확인할 출력:

| 출력 | 확인 의미 |
|---|---|
| `실행할 trade_plans가 없습니다` | planner 미실행 또는 오늘 계획 없음 |
| `PENDING/ORDERED 계획이 0개` | 오늘 실행 대상이 이미 닫힘 |
| `[RUNNER] 손실 한도 초과` | 전일 `balance_history` 총자산의 10% 기준 일일 손실 한도 차단 |
| `[STATUS] ... PENDING/ORDERED ...` | 남은 실행 대상 수 |

executor는 개별 plan에서 예외가 나도 다음 plan을 계속 처리한다. 오류가 난 plan은 다음 사이클에서 다시 관찰될 수 있다.

## 15:40 reconciler

```bash
python -m apps.trader reconciler
```

확인할 출력:

| 출력 | 확인 의미 |
|---|---|
| `15:40 대기를 건너뜁니다` | 오늘 계획이 있고 `PENDING/ORDERED`가 0개 |
| `EOD reconcile 완료` | KIS 주문 이력으로 `orders`/`executions` 정산 |
| `SELL_ONLY -> REMOVED` | 청산 완료된 SELL_ONLY universe 제거 |
| `balance_history 스냅샷 저장` | 다음 거래일 손실 한도 기준 저장 |
| 최종 `[STATUS]` | DONE/SKIPPED/PENDING 상태 확인 |

`balance_history` 저장이 건너뛰어지면 다음 거래일 손실 한도 체크가 기준 스냅샷 부재로 통과 처리될 수 있다.

## 종료 기준

오늘 계획이 정상적으로 닫힌 상태:

```text
PENDING/ORDERED = 0
DONE + SKIPPED = total_plans
```

주의할 상태:

| 상태 | 의미 |
|---|---|
| `PENDING` | 아직 주문 시도 전이거나 조건 미충족으로 다시 시도할 수 있음 |
| `ORDERED` | 주문 제출/부분 처리 이후 재시도 또는 reconcile이 필요한 상태 |
| `DONE` | plan 기준 완료 |
| `SKIPPED` | 오늘 주문하지 않기로 기록 |
| `CANCELLED` | 인터럽트 등으로 취소 처리 |

## 테스트 실행

```bash
python -m apps.trader planner --test
```

`--test`는 시간 대기를 건너뛰므로 개발 검증에 편하다. 다만 `executor --test`는 실행 대상이 계속 남으면 루프가 오래 지속될 수 있다.
