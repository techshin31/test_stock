# 환경변수 레퍼런스

근거 코드: `apps/trader/config.py`, `apps/trader/.env.example`, `apps/trader/audit.py`, `apps/trader/monitor.py`

`load_config()`는 기본적으로 `apps/trader/.env`를 읽는다. `QUANTPILOT_ENV_FILE`이 있으면 그 경로를 대신 읽는다.

```text
Path(env_file or os.getenv("QUANTPILOT_ENV_FILE", "apps/trader/.env"))
```

이미 프로세스 환경 변수에 값이 있으면 `.env`가 덮어쓰지 않는다. `load_dotenv(..., override=False)` 동작 때문이다.

## 필수 값

| 변수 | 필수 | 기본값 | 의미 |
|---|---:|---|---|
| `USER_EMAIL` | 필수 | 없음 | `users.email` 조회 기준 |
| `CREDENTIAL_ENCRYPTION_KEY` | 필수 | 없음 | DB 자격증명 복호화 키 |
| `POSTGRES_HOST` | 필수 | 없음 | DB 호스트 |
| `POSTGRES_USER` | 필수 | 없음 | DB 사용자 |
| `POSTGRES_PASSWORD` | 필수 | 없음 | DB 비밀번호 |
| `POSTGRES_DB` | 필수 | 없음 | DB 이름 |

`POSTGRES_PORT`는 필수 검증에는 없고, 없으면 `5432`를 사용한다.

`CREDENTIAL_ENCRYPTION_KEY`는 `apps/user register`로 자격증명을 저장할 때 사용한 값과 같아야 한다.

## 거래 환경

| 변수 | 필수 | 기본값 | 의미 |
|---|---:|---|---|
| `BROKER_CODE` | 선택 | `KIS` | 조회할 증권사 코드 |
| `KIS_ENV` | 선택 | `paper` | `real`이면 `REAL`, 그 외는 `PAPER`로 자격증명 조회 |
| `ALLOW_LIVE_ORDER` | 선택 | `false` | `KIS_ENV=real`일 때 실주문 허용 이중 잠금 |

현재 코드는 `KIS_ENV`를 소문자로 정규화하지 않는다. 값 비교는 `"real"`에 대해 이뤄진다.

```text
KIS_ENV=real -> environment_code=REAL
그 외 값      -> environment_code=PAPER
```

`KIS_ENV=real`인데 `ALLOW_LIVE_ORDER=true`가 아니면 `load_config()` 단계에서 종료한다. 이후 `_init()`에서도 `check_live_order_gate()`를 한 번 더 통과한다.

## 전략과 실행 간격

| 변수 | 필수 | 기본값 | 의미 |
|---|---:|---|---|
| `STRATEGY_NAME` | 선택 | `risk_neutral` | `strategies.name` 조회 키 |
| `DAILY_LOSS_LIMIT` | 선택 | `0.10` | 전일 마감 총자산 대비 손실 한도. `0 < 값 <= 1`이면 비율, `1`보다 크면 원화 고정 한도 |
| `CYCLE_INTERVAL_SEC` | 선택 | `60` | executor 사이클 간격 |

`STRATEGY_NAME`은 활성 전략이어야 한다. `fetch_strategy_params()`는 `strategies.name = 값 AND is_active = TRUE` 조건으로 조회한다.

기본 손실 한도는 전일 마감 총자산의 10%다.

```text
DAILY_LOSS_LIMIT=0.10
손실 한도 = latest balance_history.total_value * 0.10
```

예전처럼 `DAILY_LOSS_LIMIT=500000`처럼 `1`보다 큰 값을 넣으면 50만원 고정 한도로 해석한다.

## 알림과 감사 로그

| 변수 | 필수 | 기본값 | 의미 |
|---|---:|---|---|
| `SLACK_WEBHOOK_URL` | 선택 | 없음 | reconciler 종료 알림 |
| `AUDIT_LOG_PATH` | 선택 | `logs/trader_audit.jsonl` | JSONL 감사 로그 저장 경로 |
| `TRADER_SKIP_WAIT` | 선택 | `false` | 거래일/시간 대기 우회. CLI `--test`가 설정 |

Slack 알림은 실패해도 예외를 전파하지 않는다.

감사 로그 이벤트:

| event | 발생 위치 |
|---|---|
| `STARTUP` | `_init()` 성공 |
| `GATE` | 실주문 gate 확인 |
| `POSITION_SYNC` | 포지션 동기화 |
| `CYCLE_START` | executor 사이클 시작 |
| `ORDER` | 체결 수량이 있는 주문 |
| `CYCLE_END` | executor 사이클 종료 |
| `EOD_RECONCILE` | 장마감 reconcile |
| `LOSS_LIMIT_BREACH` | 일일 손실 한도 차단 |
| `ERROR` | 주요 예외 |

## 현재 README와 코드의 차이

현재 `apps/trader/config.py`는 `KIS_APP_KEY`, `KIS_APP_SECRET`, 계좌번호를 필수 환경 변수로 요구하지 않는다. Trader는 브로커 자격증명을 DB에서 조회한다.

따라서 운영 기준은 다음과 같다.

```text
apps/user/.env
  -> KIS 앱키/시크릿/계좌번호 등록

apps/trader/.env
  -> USER_EMAIL, CREDENTIAL_ENCRYPTION_KEY, DB, 전략, gate 설정
```

Trader가 필요한 DB 자격증명 조건:

| 조건 | 조회 기준 |
|---|---|
| 사용자 | `USER_EMAIL` |
| 증권사 | `BROKER_CODE` |
| 계좌 유형 | `extra.account_type = STOCK` |
| 거래 환경 | `KIS_ENV -> REAL/PAPER` |
| 활성 여부 | `user_broker_credentials.is_active = TRUE` |
