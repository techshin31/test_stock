# 리스크와 운영 상태

## 상태 코드

| 상태 | 투자자 관점 |
|---|---|
| `RUNNING` | 분석 중. 1시간 이상 남으면 다음 실행 준비 시 FAIL 처리 대상 |
| `PASS` | 결과 검증 통과. 아직 운영 universe 미발행 |
| `WARNING` | 핵심 시점 검증은 통과했지만 품질 경고 있음. 발행 가능 |
| `FAIL` | 분석/검증 실패. 운영 발행 불가 |
| `PUBLISHED` | 운영 universe에 반영 완료 |

## WARNING을 어떻게 볼 것인가

WARNING은 매수 금지 신호가 아니다. 다만 아래 항목 중 일부가 기대 기준보다 약하다는 뜻이다.

- 필수 매크로 결과 일부 누락
- 업종/기업 선정 수가 기대 범위와 다름
- 기업 계약 조건 경고
- effective date 기준 위험상태 경고
- readiness가 WARNING인 입력 사용

운영 발행 전에는 `validation_summary`의 check별 detail을 확인한다.

## 매수 차단 리스크

Analyzer는 `company_risk_states`를 두 번 사용한다.

| 위치 | 기준일 | 목적 |
|---|---|---|
| 업종 점수 | cutoff date | 업종 내 eligible LARGE 수 계산에서 차단 종목 제외 |
| 기업 선정/검증/발행 | cutoff 또는 effective date | 실제 선택 기업이 매수 차단 상태인지 검증 |

DART 이벤트 기반 위험상태 예:

- 유상증자
- 전환사채
- 신주인수권부사채
- 교환사채

## universe 발행의 의미

`analyze all --publish`는 투자 후보를 운영 상태로 바꾸는 행위다.

| universe 상태 | 의미 |
|---|---|
| `ACTIVE` | Trader가 신규 매수/보유 대상으로 사용할 수 있음 |
| `SELL_ONLY` | 기존 보유분 청산 대상. 신규 매수 후보가 아님 |
| `REMOVED` | 보유 수량이 없어 운영 universe에서 제거된 상태 |

발행 시 기존 ACTIVE였지만 이번에 미선정된 종목은 즉시 `SELL_ONLY`가 되고, 20거래일 기준 청산 기한이 계산된다.

## 감사 명령

```powershell
python -m apps.worker audit
```

감사 항목:

| 항목 | 의미 |
|---|---|
| `macro_point_in_time_violations` | 매크로 결과가 cutoff 이후 available_date를 사용 |
| `company_point_in_time_violations` | 기업 결과가 cutoff 이후 available_date를 사용 |
| `stale_running_count` | 1시간 이상 고착된 RUNNING run |
| `published_universe_mismatches` | 최신 PUBLISHED 선택 기업과 ACTIVE universe 불일치 |
| `average_monthly_turnover` | PUBLISHED run 간 종목 교체율 평균 |
