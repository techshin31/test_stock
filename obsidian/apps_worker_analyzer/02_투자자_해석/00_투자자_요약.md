# 투자자 요약

Analyzer는 매월 “현재 매크로 환경에서 유리하거나 방어력이 있는 업종”을 먼저 고르고, 그 업종 안에서 “대형주 중 분기 재무 품질이 좋은 기업”을 고른다.

결과적으로 운영 후보는 최대 10개다.

```text
최대 5개 WICS 중분류 업종
  x 업종별 최대 2개 기업
  = 최대 10개 ACTIVE universe 종목
```

## 투자 아이디어

이 시스템은 bottom-up 종목 발굴기가 아니라 top-down FA 로테이션이다.

| 단계 | 투자 의미 |
|---|---|
| 매크로 방향 | 금리, 원자재, 위험, 환율, 제조업, 한류 지표가 어느 쪽으로 기울었는지 판단 |
| 업종 관계 | 과거에 해당 매크로 변화와 같이 움직인 WICS 업종을 찾음 |
| 업종 점수 | 매크로 적합도, 업종 내 기업 재무 폭, 유동성/수용력, 리스크를 합산 |
| 기업 점수 | 선택 업종 안에서 분기 재무 품질과 개선세가 좋은 KOSPI LARGE 기업 선정 |
| 위험상태 | DART 이벤트 기반 매수 차단 종목은 후보에서 제외 |

## 무엇을 믿고 무엇을 조심해야 하나

믿을 수 있는 부분:

- `cutoff_date` 이후에 공개된 데이터는 결과 검증에서 걸러진다.
- `company_risk_states`가 effective date에 유효하면 신규 매수 후보에서 제외된다.
- Trader는 raw FA 결과가 아니라 발행된 `universe`만 사용한다.

조심할 부분:

- WARNING 상태도 발행 가능하다. WARNING은 “운영 가능하지만 품질 경고가 있다”는 뜻이다.
- 업종 관계는 과거 상관과 신뢰도 기반이다. 구조적 체제 변화는 자동으로 설명하지 못한다.
- `fa_score`는 절대 가치평가 점수가 아니라 같은 분기/모델 cohort 안의 상대 품질 점수다.

## 결과를 읽는 순서

1. `fa_analysis_runs.status_code`, `validation_summary` 확인
2. `fa_sector_results`에서 `is_selected`, `sector_score`, `reason_code` 확인
3. `fa_company_results`에서 `is_selected`, `fa_score`, `score_confidence`, `exclusion_reason_code` 확인
4. 운영 반영 여부는 `universe`의 `ACTIVE`, `SELL_ONLY`, `source_fa_company_result_id`로 확인
