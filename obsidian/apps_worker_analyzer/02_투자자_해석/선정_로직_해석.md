# 선정 로직 해석

## 1. 매크로 방향

Analyzer는 17개 매크로 신호를 읽고 각 신호를 `UP`, `DOWN`, `FLAT`으로 분류한다.

| 신호 묶음 | 예시 | 해석 |
|---|---|---|
| 원자재 | COPPER, GOLD, WTI | 경기/인플레이션/위험 선호의 가격 신호 |
| 금리 | TNX, US2Y, CPI | 금리와 물가 압력 |
| 위험 | SOX, BDRY, VIX, GPR | 반도체, 해운, 변동성, 지정학 위험 |
| FX | DXY, USDKRW | 달러와 원화 환경 |
| 제조업 | ISM_PMI, SEMIPROD | 제조업과 반도체 생산 |
| 한류 | GTREND_KPOP, GTREND_KDRAMA, KR_TOURIST | 콘텐츠/소비/관광 민감 업종 보조 신호 |

일간 신호는 20/60/120 관측치 변화, 월간 신호는 3/6/12개월 변화 또는 YoY 변화 중심으로 trend를 계산한다. trend 절댓값이 크면 `trend_strength`가 커진다.

## 2. 매크로와 업종 관계

각 매크로 변화와 WICS 업종 수익률을 맞춰 상관, beta, sample count, sign stability를 계산한다.

관계가 유효하려면 다음 기준을 통과해야 한다.

| 기준 | 기본값 |
|---|---|
| 일간/주간 관계 최소 샘플 | 104 |
| 월간 관계 최소 샘플 | 36 |
| 최소 절대 상관 | 0.15 |
| 최소 관계 신뢰도 | 0.50 |

한류 신호는 적용 가능한 업종이 제한된다. 예를 들어 K-pop/K-drama 검색 관심도는 화장품, 미디어, 호텔/레저 등 정책적으로 연결된 업종에만 기여한다.

## 3. 업종 후보

업종은 먼저 후보 풀 8개를 만든다.

| 후보 풀 | 개수 | 의미 |
|---|---:|---|
| `UP` | 5 | 상승 매크로 환경에서 수혜 가능성이 높은 업종 |
| `DOWN` | 3 | 하락 또는 위험 환경에서 방어/헤지 성격이 있는 업종 |

그 뒤 후보 업종을 `sector_score` 순으로 정렬해 최대 5개를 최종 선택한다. 후보가 부족하면 non-candidate 업종에서 fallback을 사용한다.

## 4. 업종 점수

`sector_score`는 0~100 범위다.

| 축 | 비중 | 의미 |
|---|---:|---|
| `macro_fit_score` | 45% | 현재 매크로와 업종의 과거 관계가 유리한가 |
| `company_fa_breadth_score` | 35% | 업종 구성 기업들의 재무 점수와 개선세가 넓게 좋은가 |
| `liquidity_capacity_score` | 20% | 거래대금과 대형주 후보 수가 충분한가 |
| `sector_risk_penalty` | 차감 | 커버리지, 관계 신뢰도, 구성종목 수, 시총 집중도, cohort 품질 미달 |

같은 macro category가 한 업종 점수를 과도하게 지배하지 않도록 category별 contribution cap을 적용한다.

## 5. 기업 선정

최종 업종별로 KOSPI LARGE ACTIVE 기업만 후보가 된다.

하드 필터:

| 제외 코드 | 의미 |
|---|---|
| `NOT_LARGE` | WICS 규모 코드가 LARGE가 아님 |
| `NO_QUARTER_FA` | cutoff 기준 사용 가능한 분기 FA 원장이 없음 |
| `CAPITAL_IMPAIRMENT` | 자본 훼손 또는 총자본 조건 미달 |
| `LOW_CONFIDENCE` | 점수 신뢰도 부족 |
| `LOW_FA_SCORE` | 최소 FA 점수 미달 |
| `BUY_BLOCKED` | 거래정지/상태 문제 또는 DART 이벤트 기반 매수 차단 |
| `MAPPING_ERROR` | 업종/시장/상태 매핑 문제 |

선정 정렬:

```text
fa_score desc
score_confidence desc
latest_trd_amt desc
stock_code asc
```

업종별 최대 2개 기업만 `is_selected=True`가 된다.
