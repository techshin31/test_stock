# Analyzer 다이어그램 지도

Analyzer 문서는 큰 그림과 상세 그림을 분리한다. 상세 다이어그램은 투자자가 먼저 읽을 수 있도록 한국어 의미를 앞에 두고, 코드명과 변수명은 괄호나 `코드:` 주석으로 붙인다.

| 문서 | 보는 내용 |
|---|---|
| [[analyze_all_전체흐름|analyze all 전체흐름]] | CLI부터 발행까지 전체 단계 |
| [[run_lifecycle_상세|run lifecycle 상세]] | 분석 실행 생성, 입력 재사용, 단계 실행 |
| [[macro_관계분석_상세|macro 관계분석 상세]] | 매크로 방향과 업종 민감도 계산 |
| [[sector_점수선정_상세|sector 점수선정 상세]] | 업종 점수 축, 후보 8개, 최종 5개 선정 |
| [[company_FA_기업선정_상세|company FA 기업선정 상세]] | 분기 재무 품질 점수와 업종별 top 2 기업 선정 |
| [[validation_publish_audit_상세|validation publish audit 상세]] | 결과 검증, 운영 후보 발행, 감사 |
| [[storage_lineage_상세|storage lineage 상세]] | 원천 데이터부터 실제 매매 후보까지 저장 계보 |
| [[투자자_의사결정_흐름|투자자 의사결정 흐름]] | 투자자 관점의 선정 논리 |

권장 독서 순서:

```text
투자자: 투자자 의사결정 흐름 -> sector -> company -> validation/publish
개발자: run lifecycle -> macro -> sector -> company -> storage lineage
운영자: analyze all 전체흐름 -> validation/publish -> storage lineage
```

## 다이어그램 표기 규칙

| 표기 | 의미 |
|---|---|
| `주석:` | 투자자/운영자가 바로 해석해야 하는 설명 |
| `코드:` | 실제 함수명, 변수명, 테이블명 |
| `cutoff` | 분석에 쓸 수 있는 데이터 마감일 |
| `effective` | Trader가 새 후보군을 적용할 날짜 |
| `universe` | Trader가 실제로 읽는 운영 후보군 테이블 |
