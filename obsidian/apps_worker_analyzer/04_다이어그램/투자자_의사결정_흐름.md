# 투자자 의사결정 흐름

이 그림은 투자자 관점의 의사결정 흐름이다. 개발자 관점의 함수/테이블 상세는 [[run_lifecycle_상세|run lifecycle 상세]]와 각 단계별 상세 다이어그램을 본다.

```mermaid
flowchart TB
    A["매크로 신호 17개<br/>주석: 시장 환경을 읽는 온도계<br/>예: 원자재, 금리, 위험, 환율, 제조업, 한류"] --> B["방향 판단<br/>코드: UP / DOWN / FLAT<br/>주석: 상승 압력 / 하락 압력 / 중립"]
    B --> C["업종 민감도<br/>코드: correlation, beta, confidence<br/>주석: 과거에 이 매크로 변화와 같이 움직인 업종인가"]
    C --> D["업종 매력도 점수<br/>코드: sector_score<br/>주석: 매크로 45%, 업종 내 재무 폭 35%, 유동성 20%, 리스크 차감"]
    D --> E["1차 후보 업종 8개<br/>코드: candidate_source_code<br/>주석: 수혜 후보 5개 + 방어 후보 3개"]
    E --> F["최종 업종 최대 5개<br/>코드: is_selected<br/>주석: 업종 안에 살 만한 대형주가 2개 이상 있어야 함"]
    F --> G["업종 내 기업 필터<br/>코드: hard filter<br/>주석: KOSPI 대형주, 정상 거래, 위험 공시 없음"]
    G --> H["기업 재무 품질 점수<br/>코드: fa_score<br/>주석: 현재 체력 60 + 개선세 30 + 리스크 10"]
    H --> I["업종별 상위 2개<br/>코드: industry_rank <= 2<br/>주석: 최종 후보는 최대 10개"]
    I --> J{"운영 반영할까?<br/>코드: --publish"}
    J -->|아니오| K["분석 결과만 저장<br/>주석: 투자 검토용, 실제 매매 후보군은 미변경"]
    J -->|예| L["운영 후보군 반영<br/>코드: universe ACTIVE<br/>주석: Trader가 실제 매매 계획에 사용"]
    L --> M["기존 보유 중 탈락 종목<br/>코드: SELL_ONLY<br/>주석: 새 매수 금지, 청산 계획 대상"]
```

투자자가 볼 핵심 질문:

| 질문 | 확인 위치 | 주석 |
|---|---|---|
| 왜 이 업종이 선택됐나 | `fa_sector_results.macro_contributions`, `sector_score`, `reason_code` | 매크로 환경, 업종 내 기업 체력, 유동성을 합산한 업종 매력도와 선택 사유 |
| 왜 이 종목이 선택됐나 | `fa_company_results.fa_score`, `score_confidence`, `industry_rank` | 업종 안에서 재무 품질과 점수 신뢰도가 높아 상위권에 든 종목 |
| 왜 이 종목이 제외됐나 | `fa_company_results.exclusion_reason_code`, `reason` | 재무 점수, 신뢰도, 거래 상태, 위험 상태 중 어떤 조건에서 걸렸는지 |
| 실제 매매 후보인가 | `universe.universe_status_code = ACTIVE` | Trader가 신규 매수 계획에 사용할 수 있는 운영 후보 |
| 기존 보유 청산 대상인가 | `universe.universe_status_code = SELL_ONLY` | 새 매수는 막고 기존 보유분 정리만 허용하는 상태 |
