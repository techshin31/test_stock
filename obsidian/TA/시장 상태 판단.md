# 🧠 시장 상태 판단 (Regime Detection)

> [[TA 개요|← TA 개요]]

## 개념

자동매매에서 **가장 먼저 판단해야 할 것**: 현재 시장이 추세장인가? 횡보장인가?

> 전략 선택이 잘못되면 아무리 좋은 지표도 손실로 이어진다.  
> 추세장에서는 추세 추종 전략, 횡보장에서는 역추세 전략이 유효.

---

## 권장 Regime Detection 로직

```python
if ADX > 25 and MA_slope > 0:
    regime = "상승 추세"   → MACD 추세 추종 전략
elif ADX > 25 and MA_slope < 0:
    regime = "하락 추세"   → 관망 or 공매도 전략
else:  # ADX < 25
    regime = "횡보"        → Bollinger Bands / RSI 역추세 전략
```

---

## 판단 지표 조합

### 필수 조합

| 지표 | 역할 |
|------|------|
| [[ADX]] | 추세 강도 수치 (25 기준) |
| 이동평균 기울기 (MA slope) | 추세 방향 결정 |

### 추가 옵션

| 지표 | 역할 |
|------|------|
| **Bollinger Band Width** | 밴드 수축 = 변동성 압축 → 폭발 준비 |
| **ATR (Average True Range)** | 절대 변동성 크기 측정 |

---

## 시장 상태별 전략 매핑

### 상승 추세 (ADX > 25, MA 우상향)
- 진입: [[MACD]] 골든크로스 → [[RSI]] 40~60 확인
- 보조: [[OBV]] 상승 추세 확인
- 손절: ATR 기반 → [[청산 전략]]

### 하락 추세 (ADX > 25, MA 우하향)
- 기본: 관망 or 현금 비중 확대
- 고급: 공매도 전략 (역추세 지표 사용)

### 횡보 (ADX < 25)
- 진입: [[Bollinger Bands]] 하단 터치 + [[RSI]] < 30
- 청산: 중심선(20MA) 도달 시 익절
- 손절: ATR 기반 타이트하게 설정

## 판단 오류 주의

| 오류 | 결과 |
|------|------|
| 횡보장에서 추세 추종 | Whipsaw로 손실 반복 |
| 추세장에서 역추세 | 손절 반복 |

## 자동매매 아키텍처에서의 위치

```
1단계: Regime 판단 (ADX + MA slope)   ← 여기
2단계: 방향 결정 (MACD)
3단계: 진입 확인 (RSI / OBV)
4단계: 포지션 사이징
5단계: 진입
6단계: Exit (손절/익절/신호/시간)
```
