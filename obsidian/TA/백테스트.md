# 🧪 백테스트 (Backtesting)

> [[TA 개요|← TA 개요]]

## 개념

전략을 실전에 투입하기 전 **과거 데이터로 성과를 검증**하는 필수 단계.

> 어떤 전략도 백테스트 없이 실전 투입 금지  
> 백테스트 → 포워드 테스트 → 소액 실전 → 본격 투입

---

## 필수 검증 단계

### 1. 데이터 분리 (Train/Test Split)

```
전체 데이터 → Train(70%) + Test(30%)
Train: 전략 최적화
Test:  실전 성과 예측 (미래 데이터로 검증)
```

### 2. Walk-Forward 테스트

```
구간별 반복 검증으로 과최적화 방지:

기간 1 [Train | Test]
기간 2      [Train | Test]
기간 3           [Train | Test]
...
```

- 한 구간에서만 잘 작동하면 과최적화 가능성 높음
- 모든 구간에서 안정적이어야 실전 신뢰 가능

### 3. 다양한 종목 적용

```
특정 종목에서만 잘 작동 → 과최적화
다양한 종목·섹터에서 안정적 → 범용성 높음
```

---

## 핵심 성과 지표

| 지표 | 의미 | 기준 |
|------|------|------|
| **CAGR** | 연복리 수익률 | 시장 수익률 초과 여부 |
| **MDD** | 최대 낙폭 (Max Drawdown) | 20% 이내 권장 |
| **샤프 지수** | 위험 대비 수익 | 1.0 이상 |
| **승률** | 수익 거래 비율 | 50% 미만도 수익 가능 |
| **손익비** | 평균 수익 / 평균 손실 | 2.0 이상 |

> 승률 40% + 손익비 3:1 = 기대값 양수 → 수익 전략

---

## 과최적화(Over-fitting) 방지

```
경고 신호:
- 너무 많은 파라미터 조정
- 특정 기간에서만 탁월한 성과
- In-sample 성과 >> Out-of-sample 성과
```

### 방지 방법
1. 파라미터 최소화 (단순함 유지)
2. Walk-Forward 테스트
3. 다양한 종목/시장에 적용
4. 논리적 근거 없는 파라미터 제거

## 포워드 테스팅

```
백테스트 통과 → 소액으로 포워드 테스트 (실시간 모의투자)
목적: 실제 시장 조건에서 알고리즘 안정성 확인
기간: 최소 3개월 이상 권장
```

## 트레이딩 시스템 개발 순서

```
1. 전략 설계 (→ [[트레이딩 시스템 아키텍처]])
2. 백테스트 (Train/Test)
3. Walk-Forward 검증
4. 포워드 테스트 (소액 실전)
5. 본격 투입 + 지속 모니터링
```
