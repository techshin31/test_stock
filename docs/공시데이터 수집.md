1. 정기 공시 일정 (법적 마감 기한)
상장사들은 분기마다 실적을 발표해야 할 의무가 있으며, 보통 분기 종료 후 45일 이내에 발표합니다. (12월 결산 법인 기준)

1분기 보고서 (1~3월 실적): 5월 15일까지

반기 보고서 (1~6월 실적): 8월 14일경까지

3분기 보고서 (1~9월 실적): 11월 14일경까지

사업 보고서 (1년 전체): 다음 해 3월 31일까지 (회계감사 등으로 기간이 더 김)

2. '잠정 실적'과 '확정 실적'
투자자들에게는 속도가 생명이기 때문에, 최종 보고서를 내기 전에 핵심 숫자(매출, 영업이익)만 먼저 발표하는 잠정 실적 공시가 있습니다.

삼성전자/LG전자 같은 대기업: 분기가 끝나고 보통 1~2주 안에 잠정 실적을 발표합니다. (예: 1월 초에 4분기 실적 발표)

중소형주: 잠정 실적 발표 없이 마감 기한에 임박해서 보고서 하나로 끝내는 경우도 많습니다.

3. 분석 시스템 설계 시 고려할 점 (개발자 관점)
만약 주식 분석 시스템을 구축하신다면, 데이터가 들어오는 날짜가 제각각이라는 점 때문에 다음 로직이 반드시 필요합니다.

스케줄러(Crawler/API Fetcher): 매일 특정 시간(예: 장 마감 후)에 새로운 공시가 떴는지 DART API를 통해 체크해야 합니다.

데이터 업데이트 트리거: 새로운 재무제표가 수집되면, 그 즉시 해당 종목의 PER, PBR, ROE를 다시 계산하여 DB를 갱신해야 합니다. 주가는 매일 변하고, 분모인 이익(EPS)은 분기마다 변하기 때문입니다.

결산월 확인: 대부분 12월에 회계연도가 끝나지만, 간혹 3월이나 9월에 결산을 하는 회사들도 있습니다. 시스템이 '현재가 몇 분기 데이터인가'를 판단할 때 결산월 기준으로 계산하게 만들어야 오류가 없습니다.

# 기업별 예상치 데이터 조회 방법 

주식 시장에서는 이를 **'컨센서스(Consensus)'**라고 부릅니다.

애널리스트들이 "이 회사는 이번에 이만큼 벌 것이다"라고 예측한 평균치인데, 실제 발표된 실적이 이 예상치를 뛰어넘으면 이를 **'어닝 서프라이즈(Earnings Surprise)'**라고 부르며 주가가 급등하는 촉매제가 되기도 합니다.

분석 시스템을 만드실 때 이 '예상치' 데이터를 수집하는 구체적인 방법들을 정리해 드릴게요.

1. 어디서 수집하나?
🇰🇷 한국 주식 (국내)
에프앤가이드(FnGuide) / 네이버 페이 증권: 가장 대중적인 소스입니다. 네이버 증권의 '종목분석' 탭에 나오는 컨센서스 데이터가 보통 에프앤가이드의 데이터를 기반으로 합니다.

DeepSearch / WiseReport API: 전문적인 유료 데이터 제공사입니다. 기업의 예상 실적을 데이터베이스 형태로 깔끔하게 제공합니다.

네이버 증권 크롤링: 라이브러리가 따로 없더라도 Python의 BeautifulSoup이나 Selenium을 활용해 웹 페이지에 적힌 '컨센서스' 표를 긁어오는 방식을 많이 사용합니다.

🇺🇸 미국 주식 (해외)
Yahoo Finance (yfinance): 가장 간편합니다. ticker.analyst_price_target이나 ticker.earnings_estimate 기능을 통해 분석가들의 목표가와 예상 수익을 가져올 수 있습니다.

Alpha Vantage / Finnhub: 'Earnings Calendar' API를 통해 예상치(Estimate)와 실제치(Actual)를 동시에 제공하므로 비교 로직을 짜기에 매우 좋습니다.

2. 시스템에 반영할 핵심 데이터 항목
예상치를 수집할 때는 단순히 '금액'만 가져오는 게 아니라 아래 항목들을 세트로 묶어서 저장해야 분석이 가능합니다.

EPS Estimate (주당순이익 예상치): 가장 핵심 지표입니다.

Revenue Estimate (매출액 예상치): 외형 성장도 예상대로인지 확인합니다.

Surprise (%): (실제치 - 예상치) / 예상치. 이 비율이 높을수록 시장의 충격(호재)이 큽니다.

Number of Analysts: 몇 명의 전문가가 예측에 참여했는지도 중요합니다. 1명이 예측한 것보다 20명이 예측한 평균값이 더 신뢰도가 높겠죠?

3. 분석 로직 예시 (의사 코드)
시스템이 자동으로 서프라이즈 종목을 찾는 로직은 보통 이런 식으로 구성됩니다.

💡 개발 시 주의사항: '가이던스'의 함정
실적이 예상보다 잘 나왔는데도 주가가 떨어지는 경우가 있습니다. 이는 기업이 실적 발표와 함께 발표하는 **'가이던스(향후 전망)'**가 어둡기 때문인 경우가 많습니다.

따라서 고급 시스템으로 발전시키려면 [과거 예상치 vs 실제치] 뿐만 아니라, [기업이 새로 제시한 다음 분기 예상치] 데이터도 함께 수집하는 것이 좋습니다.
