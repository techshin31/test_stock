# 자동 트레이딩 시스템 완성도 감사

기준 시각: 2026-07-22 KST  
운영 범위: PAPER 자동매매. REAL 실행·활성화는 이 감사 범위에서 금지한다.

## 완료 판정 원칙

코드가 존재한다는 사실만으로 완료로 판정하지 않는다. 각 항목은 테스트, 현재 PAPER 원장, 실행 프로세스, 공식 일별 보고서 중 하나 이상의 직접 증거가 있어야 한다. 장기간 관측이 필요한 항목은 코드가 완성돼도 `관측 중`으로 유지한다.

## 현재 핵심 차단 항목

1. 안정화 이후 체결 표본이 BUY 5건, SELL 4건으로 방향별 최소 30건을 충족하지 못했다.
2. 추세 재무장 shadow는 최소 10세션 중 1세션만 완료됐다.
3. PAPER→REAL 운영 관측기간은 1/60세션이며, FINAL·READY 일일 보고서도 1/60건이다.

5억원 원장과 패리티 차단 항목은 해소됐다. KIS 연속조회 전체 페이지에서 216개 주문행을 감사해 DB의 잘못된 체결 7건과 누락된 브로커 체결 5건을 확인했다. 547개 주문 이벤트와 201개 실제 체결을 재생한 결과 전체 23개 종목 및 현재 4개 보유종목 수량이 모두 일치했고, 5억원 시작 현금 잔차는 391,449원(0.0783%)으로 명시한 0.10% 한도 안이다.

## 다음 구현 순서

1. 매 거래일 당일 executions 무결성과 브로커 감사 미해결 0건을 유지한다.
2. BUY/SELL 각 30건까지 체결 사후분포를 자동 누적한다.
3. 10세션 shadow와 60개 완료 PAPER 세션·60개 FINAL/READY 일일 보고서를 자동 누적한다.
4. 표본 충족 후 C_CAP10·C_CAP08과 현행 규칙을 동일 스트레스 조건에서 재평가한다.
5. 모든 관측 게이트 통과 전에는 전략 변경과 REAL 실행을 계속 차단한다.

이 문서의 `증거 부족`과 `관측 중` 항목이 모두 완료 조건을 충족하기 전에는 자동 트레이딩 시스템 전체 완료로 판정하지 않는다.

## 2026-07-22 체결 스트레스 및 일일 근거 갱신

- PAPER 최종 일일 리포트는 최신 원장, 주문결과 리플레이, 체결 스트레스 결과를 표준 `latest` 경로에 자동 갱신한다.
- 체결 스트레스는 현재 원장 주문 건수와 일치해야 하며, BUY/SELL 각 30건 이상과 모든 체결 시나리오 통과를 요구한다.
- 현재 표본은 BUY 5건, SELL 4건이고 3개 시나리오 중 2개만 통과하므로 `R_TREND_REARM`은 관찰 전용으로 유지한다.
- 현재 PAPER 런타임 안전은 통과했지만 전체 시스템 완료는 과거 원장, 리플레이, shadow 10세션, PAPER 60거래일 증거가 충족될 때까지 차단한다.
- KIS 공식 연속조회 규칙의 `F` 헤더를 반영한 뒤 7월 9일 조회가 15건에서 53건으로 확대됐고, 누락 체결 5건의 가격·수량·총체결금액을 모두 복구했다. 감사 가능한 체결 근거와 리플레이 가격 커버리지는 100%다.
- DB에 없던 브로커 매도 5건도 직접 확인해 원장에 감사 이벤트로 반영했다. 이후 전략 계보와 잘못된 DB 체결 7건까지 대사해 현재 및 전체 종목 수량 일치율을 100%로 복구했다.

## 2026-07-22 원장·패리티 최종 대사

- 초기 `risk_neutral` 거래를 `aggressive`와 같은 PAPER 계좌 계보로 포함하고, 7월 3일부터 22일까지 KIS 주문 216행을 자동 감사한다.
- DB가 FILLED로 잘못 기록한 012510 BUY 7건(합계 1,378주)은 KIS 체결수량 0을 근거로 분석 원장에서만 `EXPIRED_UNFILLED` 처리했다. 원본 DB와 계좌는 변경하지 않았다.
- 최신 원장은 547개 주문 이벤트와 201개 실제 체결을 포함한다. 전체 23개 종목과 현재 보유 4개 종목 수량이 모두 정확히 일치하고, 시작재고·수량 조정항목은 0이다.
- 5억원 직접 재생의 종료 현금 차이는 391,449원(0.0783%)으로 명시한 0.10% 허용한도 500,000원 이내다. 보정 후 종료 현금과 보유수량은 정확히 일치한다.
- C_CAP10과 C_CAP08은 수익률 -10%, MDD -20%, 회전율 30배 한도에서 세 체결 시나리오를 모두 통과했다. 다만 BUY 5건, SELL 4건의 작은 표본이므로 실제 주문 규칙에는 반영하지 않는다.
- 현재 완료 차단 항목은 방향별 30건 체결 표본, shadow 10세션, PAPER 60개 완료 세션과 해당 세션의 FINAL/READY 보고서 60건이다. REAL 실행 승인은 계속 거부한다.

## 2026-07-22 일일 증거 연속성 강화

- 시스템 준비도는 운영 로그의 15:30 KST 완료 세션 날짜를 직접 계산하고, `latest.json`이 가장 최근 완료 세션과 정확히 같은 날짜인지 확인한다.
- 완료된 모든 PAPER 세션에 동일 날짜의 `FINAL`·`READY` 일일 JSON이 있어야 하며, 누락·잘못된 모드·파일명/날짜 불일치는 별도 커버리지 게이트로 차단한다.
- 전체 완료에는 운영 세션 60일뿐 아니라 해당 세션과 교집합인 FINAL/READY 일일 보고서도 60건이 필요하다. 장중 현재 날짜는 15:30 전까지 완료 세션으로 세지 않는다.
- EOD 보고서 생성 뒤 준비도 감사가 실패하면 차단 JSON을 남기고 작업 자체를 실패 처리한다. 스케줄러는 성공으로 기록하지 않고 5분 간격으로 재시도한다.
- `run_scheduler.bat --live`는 기존 KPI 게이트보다 먼저 전체 시스템 준비도를 재계산한다. 직접 `run_live_trader.py --live` 주문 경로도 브로커 초기화 전에 같은 완료 조건을 요구한다. 읽기 전용 실계좌 스냅샷과 명시적 비상 청산은 별도의 안전 경로로 유지한다.

## 2026-07-22 런타임·EOD 증거 범위 강화

- 장중 PAPER 안전 판정은 최신 데이터가 기대 종목 전체에 존재하고 stale/missing/dependency 오류와 보유종목 stale이 모두 0건이어야 한다.
- 보유종목 위험평가 완료율은 100%여야 하며, 대시보드와 실제 주문 상태에 미종결 주문이 없어야 한다.
- 최근 런타임 오류와 진입 차단기가 활성화돼 있으면 PAPER 안전 및 REAL 사전 게이트를 차단한다.
- 일일 보고서는 `FINAL`·`READY` 문자열뿐 아니라 데이터 신선도, 위험점검, 주문 대사, 운영 무결성이 각각 100%이고 미종결 주문이 0건인 경우에만 완료 증거로 센다.
- EOD 완료 뒤 정식 분석 아티팩트도 `paper_system_report/latest.json`과 날짜별 스냅샷으로 원자적으로 갱신한다. 준비도 또는 아티팩트 갱신이 실패하면 EOD 작업을 실패 처리해 기존 재시도 경로로 보낸다.
- `run_scheduler.bat`는 PAPER·DRY_RUN·SIMULATE를 bounded supervisor로 실행한다. 비정상 종료는 최대 5회, 기본 30초 간격으로 복구하지만 안전거부 exit 2는 재시도하지 않는다. REAL은 어떤 exit code에서도 자동 재시작하지 않고 새 수동 게이트 실행을 요구한다.
- 무주문 복구 자기진단은 브로커를 로드하지 않은 하위 프로세스를 의도적으로 1회 실패시킨 뒤 두 번째 실행의 정상 종료를 검증한다. 현재 증거는 exit `[1, 0]`, 주문 권한 `DENIED_BY_DESIGN`, REAL 자동복구 `false`다.
- 기존 PAPER scheduler PID 2644는 재시작하지 않고 숨김 supervisor가 무중단 연결했다. 준비도는 scheduler와 supervisor의 PAPER 범위 잠금, 연결 이벤트의 대상 PID 일치를 함께 검사하며 현재 안전 검사는 12/12다.
