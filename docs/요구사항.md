# 📌 주식 분석 및 예측 서비스 요구사항 정의서 (Stock Analysis & Prediction Service)

---

# 1. 서비스 개요

주식 시장 데이터를 기반으로  
- **기초 분석(펀더멘털)**  
- **기술적 분석(차트/인디케이터)**  
- **뉴스/RAG 기반 텍스트 분석**  
- **머신러닝/딥러닝 기반 예측 모델**  
- **LLM 기반 종합 리포트**  
를 제공하는 종합 투자 인사이트 플랫폼.

사용자는 웹/앱에서 주식 종목, 차트, 분석 리포트, 예측 결과를 확인할 수 있음.

---

# 2. 핵심 기능 요구사항

## 2.1 종목 검색 및 기본 정보 조회
- 종목 검색(티커/이름)
- 기업 개요(업종, 시가총액, PER, PBR 등)
- 재무제표 제공(매출, 영업이익, 부채비율 등)
- 배당 정보 및 과거 배당 기록

---

## 2.2 실시간 시세 및 차트
- 실시간 가격(WebSocket)
- 1분/5분/일/주/월봉 차트 제공
- OHLCV 데이터 제공
- TradingView 또는 직접 구축한 차트 모듈 사용
- 공매도/거래량/기관·외인 수급 표시

---

## 2.3 기술적 분석 기능
- 주요 인디케이터 제공:
  - 이동평균(SMA, EMA)
  - MACD
  - RSI
  - Bollinger Bands
- 차트 위 지표 시각화
- 커스텀 인디케이터 추가 기능(선택)

---

## 2.4 뉴스 및 이벤트 분석
- 종목 관련 뉴스 자동 수집
- 뉴스 감성 분석(Sentiment Analysis)
- 주요 이슈 요약(LLM Summarization)
- Earnings Calendar / FOMC 일정 표시
- 급등락 원인 분석 자동화

---

## 2.5 LLM 기반 종합 분석 리포트
- 종목 분석 리포트 자동 생성
- 뉴스 요약 + 재무 분석 + 기술적 분석 결합
- “이 종목을 주목해야 하는 이유” 자동 생성
- 매수/매도 판단은 법적 문제로 제공 X  
  → 대신 “리스크 요인”과 “Opportunity” 중심 제공

---

## 2.6 머신러닝/딥러닝 예측 모델
- 가격 예측(LSTM, Transformer, XGBoost 등)
- 상승 확률(Probability of Rise) 제공
- 단기/중기/장기 타임프레임별 예측
- 모델 신뢰도 Score 표시

---

## 2.7 사용자 포트폴리오 분석
- 보유 종목 자동 연동(증권사 API)
- 수익률, 리스크 분석
- 종목간 상관관계 분석(Heatmap)
- 포트폴리오 최적화(Mean-Variance Optimization)

---

## 2.8 알림 및 Noti 서비스
- 주요 뉴스 발생 시 알림
- 급등/급락 알림
- 타겟 가격 도달 알림
- 예측 모델 결과 변화 알림
- 잔여 리스크 경고(변동성 급증 등)

---

# 3. 시스템 요구사항 (Technical Requirements)

## 3.1 데이터 파이프라인
- 실시간 데이터: WebSocket 또는 Kafka 기반
- 과거 가격 데이터: TimescaleDB
- 뉴스 데이터: 크롤링/뉴스 API
- 데이터 수집 자동화(Cron, Airflow 등)
- 데이터 검증 로직(결측치 처리, 오류 감지)

---

## 3.2 백엔드 API
- FastAPI or Django REST Framework
- 시세 API
- 인디케이터 계산 API
- 예측 모델 API
- LLM 분석 API
- 포트폴리오 분석 API
- 인증 시스템(JWT)

---

## 3.3 모델 서빙
- ML 모델 서버(TensorRT, ONNX, TorchServe)
- LLM/Agent 서버(LangGraph 기반)
- GPU 추론 환경(Optional)
- 모델 모니터링(Metrics/Latency/Error Rate)

---

## 3.4 웹/앱 프론트엔드
- 웹: React
- 앱: React Native + Expo
- 차트: TradingView / Lightweight Charts
- 알림 Push: Firebase / Expo Push

---

## 3.5 인프라 및 운영
- Docker, Kubernetes 기반 배포
- 모니터링: Grafana + Prometheus
- 로그 수집: Loki or ELK
- CI/CD: Github Actions
- 보안: SSL, 각종 인증, 보안 점검

---

# 4. 비기능 요구사항 (NFR)

## 4.1 성능
- 실시간 가격 지연 < 300ms
- LLM 리포트 생성 < 3초
- 예측 모델 응답 < 1초

## 4.2 안정성
- 고가용성(HA) 구성
- 실시간 데이터 장애 자동 복구
- 트래픽 급증 대비 Auto Scaling

## 4.3 보안
- OAuth2 / JWT 기반 인증
- 민감 데이터 암호화
- CSRF/XSS/SQL Injection 방지

# 5. 사용자 시나리오 (Use Case)

## UC1 — 종목 검색 및 분석
사용자는 종목 검색 → 기본 정보 → 차트 → 지표 → 분석 리포트를 확인한다.

## UC2 — 뉴스 기반 투자 인사이트
사용자는 특정 종목 뉴스 요약/감성 결과를 본다.

## UC3 — 가격 예측 & 모델 신뢰도 확인
사용자는 “상승 확률 + 예측 가격 + Confidence Score”를 확인한다.

## UC4 — 종목 알림
종목 이벤트 발생 시 즉시 Push 알림을 받는다.

## UC5 — 포트폴리오 진단
보유 종목의 리스크, 분산도, 상관계수 등을 분석한다.
