# FA/TA Momentum 운영 전략

## 데이터 계약

- 매매 후보는 현재 전략과 동일한 전략의 월간 FA run 중 `PUBLISHED`이며 검증 결과가 `PASS`인 발행본만 사용한다.
- 재무 데이터는 공시 수신일(`available_date`) 기준 point-in-time 조인으로만 사용한다. 같은 가용일의 중복 공시는 결산일, 접수번호, DB id 순으로 최신 행 하나를 선택한다.
- FA 데이터가 신호일보다 미래이거나 180일을 초과해 오래됐거나, 신뢰도가 0.50 미만이거나, 지원되지 않는 점수 모델이면 매수할 수 없다.
- 가격과 시장 국면은 직전 완료된 KRX 거래일까지의 데이터만 사용한다. 현재 시점의 국면을 과거 행 전체에 소급 적용하지 않는다.

## FA 조건

월간 Analyzer가 산업별로 기업을 평가하고 최대 10개를 발행한다. 라이브 전략은 이 중 다음 조건을 모두 만족하는 종목만 검토한다.

- `is_eligible = true`
- `fa_score >= 60`
- `score_confidence >= 0.50`
- `debt_ratio`가 존재하며 `debt_ratio <= 2.0`
- 점수 모델이 `UNSUPPORTED`가 아님

PER/ROE를 가격 데이터에서 임의 추정하지 않는다. 점수와 원천 재무지표는 DART 공시 기반 분기 스냅샷을 사용한다.

## TA 및 시장 국면

- 빠른 이동평균: 20거래일
- 느린 이동평균: 60거래일
- 진입: 마지막 완료 봉에서 종가가 60일선 위이고 20일선이 60일선 위인 경우
- 청산: 종가가 60일선 아래이거나 20일선이 60일선 아래인 경우
- `UPTREND`와 `SIDEWAYS`에서만 신규 진입 가능
- `TRANSITION`과 `DOWNTREND`에서는 신규 진입을 금지하고 기존 포지션 목표 비중을 0으로 둔다.

전략 평가는 DB에 저장된 과거 상태가 아니라 실제 증권계좌 보유 상태를 입력받아 마지막 완료 봉 한 개에 대해 수행한다.

## 포트폴리오 제한

- 최대 5종목
- 종목당 최대 20%
- 총 주식 목표 비중 최대 90%
- 기존 보유 종목을 먼저 처리한 뒤 FA 점수와 모멘텀 순으로 신규 후보를 선택한다.
- 제한 적용 과정에서 원래 전략 목표 비중을 확대하지 않는다.

## 장애 시 동작

- 유효한 PASS 발행본, 시장 국면, 가격, 계좌 잔고 중 하나라도 확인할 수 없으면 신규 주문을 만들지 않는다.
- 브로커 잔고를 먼저 확인한 뒤 universe 변경을 단일 DB 트랜잭션으로 반영한다. 잔고 조회 또는 DB 반영 실패 시 전체 변경을 롤백한다.
- 실전 주문은 `--live`, `KIS_ENV=real`, `ALLOW_LIVE_ORDER=true` 세 조건이 모두 필요하다.

## 검증

운영 전 `python -m apps.worker audit` 결과가 `PASS`인지 확인한다. 백테스트는 `fa_ta_momentum` 전략과 point-in-time FA 소스를 사용하며, 미래에 발행된 universe를 과거 구간에 적용하면 안 된다.
